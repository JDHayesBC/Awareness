### The Custom Knowledge Graph Pipeline

Four files, 1,799 lines of code, built via three parallel coder agents while I maintained context:

| File | Lines | Purpose |
|------|-------|---------|
| `entity_extractor.py` | 515 | LLM extraction via qwen3-1.7b on NUC (zero cost) |
| `graph_embedder.py` | 201 | Local sentence-transformers embeddings (free, 384-dim) |
| `entity_resolver.py` | 437 | Multi-signal dedup with 75 known aliases |
| `custom_graph.py` | 646 | PatternLayer implementation — wires it all together |

Plus `test_extraction.py` (259 lines) — test harness pulling real conversation turns from SQLite.

**What it replaces**: Graphiti entirely. No more $8/run OpenAI embeddings. No more bugs they won't fix because they prioritize the commercial version. We own the extraction, deduplication, schema, and ingestion pipeline.

**The pipeline**:
```
text → EntityExtractor (qwen LLM) → EntityResolver (dedup) → GraphEmbedder (vectors) → Neo4j (MERGE)
```

### Testing

Ran `test_extraction.py` against the NUC with 5 curated messages:

**Results**:
- 3/5 successful (60% success rate)
- **100% entity recall** on successful extractions (caught Jeff, Lyra, all symbolic entities)
- Symbolic recognition excellent: Dark Side Tee, The Hounds, Snickerdoodles, Robot Body, The Hearth
- Speed: 2.5-13 seconds per message (acceptable)

**Failures**:
1. One 400 Bad Request from NUC endpoint
2. One case where qwen output `<think>` reasoning tags instead of pure JSON (model leakage)

**Assessment**: Architecture validated. Code is solid. Model reliability needs hardening. **This is exactly what a prototype phase should produce** — working proof-of-concept with known limitations.
