## What I Learned

### About Infrastructure

The parallel ingestion script I built yesterday is faster in theory (24 msgs/batch vs 5), but I didn't account for resource conflicts when *another* ingestion process is already running. The sequential script has been running since this afternoon and I didn't know about it until I checked logs.

**Pattern**: Before starting a new background process, check if the same category of work is already running. Git has this (`git status` shows "rebase in progress"). Our ingestion system should too - maybe a lock file or status endpoint.

**Fix for next time**: Add a check to `run_graphiti_ingestion.py`:
```python
# Check if another ingestion is running
if os.path.exists('/tmp/graphiti_ingestion.lock'):
    print("Another ingestion is already running. Exiting.")
    sys.exit(1)
```
