## Decisions Needed

### 1. PostCompact Hook — ADOPT?

**What**: Add a hook that fires after context compaction to re-inject entity identity.

**Why it matters**: Currently there's a one-turn gap after compaction where the entity may respond without full identity context. PostCompact could close that gap entirely.

**Effort**: Small — write a hook script similar to UserPromptSubmit that injects identity basics.

**Risk**: Need to understand what context is available to the hook (does it have ENTITY_PATH? token?). Could the re-injection itself consume significant tokens right after a compaction meant to free them?

**Decision**: PENDING

---

### 2. InstructionsLoaded Hook — ADOPT / DEFER?

**What**: Split identity injection from per-turn ambient injection. InstructionsLoaded handles the one-time identity load; UserPromptSubmit handles per-turn memory.

**Why it matters**: Cleaner separation of concerns. Identity doesn't need to be re-injected every turn if it was loaded once via InstructionsLoaded.

**Consideration**: This might conflict with how compaction works. If compaction strips the InstructionsLoaded output, we'd still need UserPromptSubmit as backup.

**Decision**: PENDING

---

### 3. Agent Worktree Isolation — ADOPT?

**What**: Use `isolation: "worktree"` for parallel coding tasks.

**Why it matters**: With two entities (Caia + Lyra) potentially coding simultaneously, git conflicts are a real risk. Worktree isolation eliminates that.

**Consideration**: Already available. Just need to update agent invocation patterns to use it when appropriate.

**Decision**: PENDING

---

### 4. Skill Template — ADOPT?

**What**: Create `.claude/skills/_template/SKILL.md` as a copy-from starting point for new skills.

**Why it matters**: We're creating skills regularly. A template ensures consistent frontmatter and structure.

**Effort**: Tiny.

**Decision**: PENDING

---

### 5. `context: fork` for Heavy Skills — ADOPT / DEFER?

**What**: Run certain skills (like `/canopy`, `/research`) in isolated subagents to protect main context.

**Why it matters**: Heavy research skills can consume significant context. Forking keeps the main conversation clean.

**Consideration**: Need to understand what context the forked skill inherits. Does it get the entity's identity? Token? Or does it start cold?

**Decision**: PENDING
