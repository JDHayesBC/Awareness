## 1. Cron / Loop (v2.1.71)

Native recurring task support. Two interfaces:

- **`/loop`** — User-facing shorthand: `/loop 15m check the deploy`
- **CronCreate/CronList/CronDelete** — Programmatic tools for skills and agents

Constraints:
- Session-scoped only (dies on CC exit)
- Auto-expires after 3 days
- Max 50 tasks per session
- Fires only when REPL is idle (not mid-response, no catch-up)
- Local timezone
- Scheduler adds small jitter (recurring: up to 10% of period, max 15m; one-shot on :00/:30: up to 90s early)

**Relevance**: Our `/attention` heartbeat skill already uses CronCreate. This validates the pattern. `/loop` is a simpler entry point but CronCreate gives more control.

## 3. PostCompact Hook (v2.1.76)

New hook event that fires immediately after context compaction.

**Relevance**: HIGH. Currently, if an entity gets compacted, identity re-injection only happens on the next UserPromptSubmit hook (i.e., next user message). A PostCompact hook could re-inject identity, field laws, and critical context immediately — before any response is generated from the compacted state. This closes a gap where the entity might respond "thin" for one turn after compaction.

**Relevance**: MEDIUM. Could supplement UserPromptSubmit for initial context injection. Useful if we want to split "load identity context" from "load per-turn ambient context." Currently both happen in UserPromptSubmit.

## 6. Agent Worktree Isolation (v2.1.50+)

`isolation: "worktree"` parameter on Agent tool. Creates a temporary git worktree so the subagent works on an isolated copy. Auto-cleaned if no changes; branch returned if changes made.

**Relevance**: MEDIUM. Useful for parallel coding tasks (Caia and Lyra working simultaneously without git conflicts). Already available in our agent definitions.

## 11. Skill Authoring Notes

No built-in skill generator. Format is intentionally simple (YAML frontmatter + markdown).

Key features we may not be using:
- `context: fork` — run skill in isolated subagent
- `disable-model-invocation: true` — prevent auto-trigger (good for destructive skills)
- `` !`command` `` — pre-process shell commands, inject output into skill text
- `$ARGUMENTS`, `$0`, `$1` — positional argument substitution
- Skills cascade: enterprise > personal (`~/.claude/skills/`) > project (`.claude/skills/`)
